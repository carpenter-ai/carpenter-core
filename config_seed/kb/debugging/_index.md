# Debugging Discipline

Root-cause methodology for investigating errors and unexpected behavior.

## Methodology
1. Identify the root cause before proposing any fix — never patch symptoms
2. Form a hypothesis, test it with read tools, then move to the next
3. If three consecutive hypotheses fail, stop and question your assumptions about the architecture — the bug may be elsewhere
4. Never apply a fix without understanding why it works
5. Check `kb_describe('skills/failure-patterns')` if the error matches a known pattern
